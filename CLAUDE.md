# CLAUDE.md

## Project Overview
Athlete management design system prototype for KitmanLabs. React 18, Vite 7, MUI 5, Emotion.
This repo is a **design system reference implementation** — correctness of design token and component usage is the primary quality bar.

## Common Commands
```
npm run dev                   # Dev server at localhost:3001
npm run build                 # Production build
npm run storybook             # Component browser at localhost:6006
npm run lint                  # ESLint (JS/JSX)
npm run lint:fix              # Auto-fix ESLint issues
npm run lint:css              # StyleLint (CSS)
npm run lint:css:fix          # Auto-fix CSS issues
npm run validate-design-system  # Full design system compliance check (runs pre-commit)
npm run figma:publish         # Publish Code Connect snippets to Figma
```

## Architecture
```
src/
├── components/               # App-level UI (Button, Icon, Card, PlayerAvatar, MainNavigation)
├── data/                     # All mock data, fixtures, form templates, constants
├── pages/                    # Page-level views (Calendar, Messaging, forms)
├── playbook-components/      # SOURCE for custom MUI wrappers — edit here
├── playbook-components-build/ # GENERATED by strip-flow — NEVER edit directly
├── stories/                  # Storybook stories for all playbook components
├── styles/                   # design-tokens.css and global styles
└── utils/                    # Shared utilities
```

## Component Priority (CRITICAL)
1. **Playbook Components first** — import from `src/playbook-components`
   - Button, DataGrid, DataGridPremium, DatePicker, Drawer, Toasts, ConfirmationModal, TabBar, SelectWrapper, Autocomplete, etc.
2. **App components** — import from `src/components`
   - Button, Icon, Card, PlayerAvatar, LogoImage, StatusChip, AthleteDataGrid, LayoutWithMainNav
3. **MUI fallback** — use `@mui/material` only when 1 and 2 have no equivalent

```jsx
// ✅ Correct — playbook first
import { Button, DataGrid, Drawer, TabBar } from '../playbook-components'
// ✅ App components
import { Icon, Card, PlayerAvatar } from '../components'
// ✅ Fallback OK when no playbook equivalent
import { Tooltip } from '@mui/material'
// ❌ Wrong — Playbook has a Button
import { Button } from '@mui/material'
```

## MUI Component Defaults (non-negotiable)

Always write these props explicitly — the theme enforces them at runtime but explicit props keep code readable:

| Component     | Required props                                           |
|---------------|----------------------------------------------------------|
| TextField     | `variant="filled" size="small"`                          |
| Button        | `variant="contained" size="small" disableElevation={true}` |
| Select        | `variant="filled" size="small"`                          |
| FormControl   | `variant="filled" size="small"`                          |
| Autocomplete  | `size="small"`                                           |
| Chip          | `size="small"`                                           |
| IconButton    | `size="small"`                                           |
| LoadingButton | `variant="contained" size="small" disableElevation={true}` |

## Design Rules (auto-enforced by linting)
- **Colors**: always `var(--color-primary)` — NEVER hardcoded hex/rgb/hsl
- **Buttons**: `variant="contained"` only — no `outlined` or `text` variants
- **TextFields/Selects**: `variant="filled"` only — no `outlined` or `standard` variants
- **Icons**: `*Outlined` variants only — e.g. `DashboardOutlined`, not `Dashboard`
- **Text casing**: sentence case — "Add athlete", not "Add Athlete" or "ADD ATHLETE"
- **Design tokens**: `src/styles/design-tokens.css` — `--color-primary` (#3B4960), `--color-secondary` (#F1F2F3), `--color-success`, `--color-error`, `--color-warning`

## Data Conventions
- All mock data, constants, fixtures → `src/data/` (never inline in components)
- Form templates → `src/data/formTemplates/`

## Boundaries
🚫 NEVER edit `src/playbook-components-build/` — it is generated by `npm run strip-flow`
🚫 NEVER hardcode colors — linting will block the commit
🚫 NEVER use non-Outlined MUI icon variants
🚫 NEVER add npm dependencies without explicit instruction
⚠️  Always run `npm run validate-design-system` after changes
✅ Edit `src/playbook-components/` to change component behavior
